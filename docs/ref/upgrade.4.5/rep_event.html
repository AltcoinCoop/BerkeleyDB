Release 4.5: Replication events

Two informational returns from the DB_ENV->rep_process_message method found in previous releases of Berkeley DB have been changed to events. The DB_REP_NEWMASTER return from DB_ENV->rep_process_message is now the DB_EVENT_REP_NEWMASTER value to the DB_ENV->set_event_notify method callback, and the DB_REP_STARTUPDONE return from DB_ENV->rep_process_message is now the DB_EVENT_REP_STARTUPDONE value to the DB_ENV->set_event_notify callback.

Applications should update their handling of these events as necessary.
